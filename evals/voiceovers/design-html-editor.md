# design-html-editor — edit local HTML in a dedicated Design space

The Design space is additive: Browser, OpenCode, Cloud, authentication, providers, and existing artifact behavior remain unchanged.

1. I open an existing iPolloWork task and see a new Design button beside the current right-side tools, while Browser and the rest of iPolloWork stay exactly where they are.

2. I open Design and choose a local HTML file from this workspace; the page renders inside the dedicated Design space without turning an ordinary web page into an editable site.

3. I switch on Edit page and click a heading in the rendered HTML; the element is outlined and a small floating toolbar puts text, font size, and color directly above the selection.

4. I choose Edit text and change the heading from the compact popover, or double-click the words and type directly on the page just like editing a slide.

5. I select an image and replace it from my computer or paste a new image URL; the replacement appears in place while the advanced inspector stays one click away.

6. I open the redesigned properties panel, try a text preset, font size, alignment, and color, undo an unwanted change, then save and reopen without changing Browser, chat, OpenCode, or Cloud.
