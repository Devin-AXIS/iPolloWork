# 可编辑 PPTX 导出试点设计

## 目标

将设计面板中的幻灯片 PPTX 导出从“整页截图加少量叠加对象”改为“HTML 元素逐个映射为 PowerPoint 原生对象”。首个交付目标是让融资路演类 HTML 幻灯片中的普通文字、图片与基础形状在 PowerPoint 中可继续编辑，同时让现有模板的复杂视觉局部保持可见。

## 非目标

- 本阶段不把 HTML 编辑器重写为 Canva 式的独立结构化画布。
- 本阶段不把 CSS/JavaScript 动画自动转换成 PowerPoint 动画。
- 本阶段不承诺任意 HTML/CSS 都能生成 100% 原生可编辑 PPTX。
- 本阶段不删除或改写用户现有的模板与导出文件。

## 导出模型

导出器以每张 slide 的计算后布局为输入，按视觉堆叠顺序生成 PowerPoint 对象：

1. `img` 元素输出为原生图片对象，位置和尺寸从 DOM 矩形换算为 16:9 PowerPoint 坐标。
2. `h1` 至 `h6`、`p`、`li` 与明确标注的 `data-ipw-text` 元素输出为原生文本框。导出字体、字号、字重、斜体、颜色、不透明度、对齐、行距和字符间距。
3. 没有不受支持视觉属性的容器输出为原生矩形或圆角矩形，导出纯色填充、完整边框和可映射的外阴影。
4. 不能忠实映射的元素输出为其自身边界内的 PNG 图片，而不是整页 PNG。局部栅格图在其余原生对象之下或之上插入，以保持原网页的层叠顺序。

导出产物不再为每张 slide 无条件插入整页背景截图。只有页面自身的纯色背景会输出为幻灯片背景或完整矩形；页面背景包含复杂视觉时，才使用该背景元素的局部回退图片。

## 兼容性规则

以下元素必须原生导出：

- 纯文本段落、标题和列表；文本容器不可包含未拆分的复杂富文本子树。
- 标准 `<img>` 图片。
- 背景为纯色、四边边框一致、无复杂视觉属性的 `div`、`article`、`section`、`aside`、`header`、`footer`、`main`。

以下元素必须以局部 PNG 回退：

- CSS 渐变、`background-image`、滤镜、毛玻璃、混合模式、遮罩、裁剪路径与 CSS `transform`。
- SVG、canvas、video、伪元素生成的视觉、复杂边框或多重/内阴影。
- 文本渐变、文字阴影、旋转/滤镜文本、含无法独立重建的富文本样式的文本。

当一个容器降级为图片时，后代不再重复输出为原生对象，以免出现底图文字和原生文字重影。导出器必须返回每页的原生对象数与回退区域数，供界面提示使用。

## 图层与布局

导出器以 DOM 的文档顺序作为默认 z-order，先添加低层对象再添加高层对象。对于一个回退元素，导出器将它视为一个原子视觉区域并跳过其后代；对于原生容器，继续访问其后代以生成文本、图片和子形状。

所有坐标基于当前 slide 的 `getBoundingClientRect()`，按 13.333 × 7.5 英寸换算。所有导出的对象必须被裁剪在 slide 边界内；完全在画布外或尺寸小于 1 像素的元素不导出。

## 用户体验

PPTX 导出确认文案改为“可编辑优先导出”。确认框说明：普通文字、图片和基础形状会保留为 PowerPoint 原生对象；复杂效果会按局部图片保留。导出成功提示显示“原生对象 N 个，局部视觉回退 M 处”。若没有发生回退，提示不显示回退信息。

## 测试与验收

单元测试必须覆盖：

- 文本、图片、纯色圆角形状的原生转换条件与坐标。
- 渐变/滤镜/变换元素的局部回退分类。
- 父元素回退后跳过后代，避免重复对象。
- 整页截图不再是默认导出对象。

端到端验证使用现有融资路演模板或同结构测试页生成 PPTX，并检查 OOXML：每页至少含原生文本框；只在出现复杂视觉时含局部图片；不得有铺满 13.333 × 7.5 英寸的默认截图；所有原生文本可在 Office 中选中编辑。保留现有 PDF 导出行为不变。

## 后续阶段

当逐元素导出稳定后，再引入独立的结构化画布文档模型，使网页画布、模板、AI 生成和 PPTX 导出都读写同一组文本、图片、形状、图表和分组对象。CSS 动画继续通过独立视频导出通道生成 MP4/GIF；仅手动定义的有限动画可映射为 PowerPoint 动画。
