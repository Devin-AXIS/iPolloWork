# iPolloWork 中文问答知识库（100 问）

这是一份待产品与工程复核的知识库底稿，暂不加入公开导航。每条内容均
采用“直接回答、适用范围、能力状态、依据”的统一结构。Work、Code、
Create、Video 在这里作为检索分类使用；正式上线前仍需确认它们的最终
中英文产品名称。

## 一、产品定位

<a id="faq-001"></a>
### 001. iPolloWork 是什么？

**简答：** iPolloWork 是一个本地优先的可视化 AI 工作台，Agent 可以在同一工作区中处理代码、文件、浏览器任务、文档、演示文稿、网站、设计和视频。

**范围 / 状态：** 全产品 · Live

**依据：** [开始使用](/start-here/get-started)

<a id="faq-002"></a>
### 002. iPolloWork 适合哪些人使用？

**简答：** 它适合希望让 Agent 实际执行任务、同时保留文件控制权和结果编辑能力的个人、开发者、创作者及团队；团队管理能力由 iPolloWork Cloud 提供。

**范围 / 状态：** 本地与 Cloud · Live

**依据：** [产品说明](/start-here/get-started)

<a id="faq-003"></a>
### 003. iPolloWork 只是一个聊天机器人吗？

**简答：** 不是。聊天是下达目标的入口，Agent 还可以读取和修改文件、运行命令、使用浏览器与扩展，并把结果留在可继续编辑的工作区中。

**范围 / 状态：** 桌面端与远程 Worker · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-004"></a>
### 004. iPolloWork 和 Codex、Claude Code 是什么关系？

**简答：** iPolloWork 将自己定位为本地优先的 Codex/Claude Code 替代选择，但产品范围不止代码，还覆盖办公、设计和视频等可编辑结果。

**范围 / 状态：** 产品定位 · Live

**依据：** [开始使用](/start-here/get-started)

<a id="faq-005"></a>
### 005. 不连接 iPolloWork Cloud 可以使用吗？

**简答：** 可以。本地 iPolloWork 不依赖 Cloud；Cloud 主要增加组织、共享资源、托管 Worker、权限和企业管理能力。

**范围 / 状态：** 本地端 · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-006"></a>
### 006. 使用本地版必须注册账号吗？

**简答：** 本地工作区不要求 iPolloWork Cloud 账号；连接 Cloud 组织、共享资源或使用托管能力时需要登录对应的 Cloud 账号。

**范围 / 状态：** 本地与 Cloud · Live

**依据：** [开始使用](/start-here/get-started)

<a id="faq-007"></a>
### 007. Work、Code、Create、Video 是四套独立应用吗？

**简答：** 当前更适合把它们理解为同一工作区里的四类任务入口，而不是彼此隔离的四套运行时；它们共享会话、工作区文件、模型和扩展能力。

**范围 / 状态：** 官网信息架构 · 待确认命名

**依据：** [产品说明](/start-here/get-started)

<a id="faq-008"></a>
### 008. iPolloWork 可以生成哪些类型的结果？

**简答：** 当前产品说明覆盖代码、文档、网站、演示文稿、设计和视频；具体格式和编辑能力取决于对应的工作区表面与已启用扩展。

**范围 / 状态：** 全产品 · Live

**依据：** [开始使用](/start-here/get-started)

<a id="faq-009"></a>
### 009. AI 生成的结果还能继续编辑吗？

**简答：** 可以。可编辑结果是 iPolloWork 的核心定位，代码和文件保留在工作区，设计与视频也有对应的编辑表面，而不是只返回一段聊天文本。

**范围 / 状态：** Code、Create、Video · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-010"></a>
### 010. iPolloWork 是开源软件吗？

**简答：** iPolloWork 是 source-available（源码可用）软件，不应描述为 OSI 认可的开源软件；可用范围和商业使用条件以仓库 LICENSE 为准。

**范围 / 状态：** 授权政策 · Policy

**依据：** [iPolloWork License](https://github.com/Devin-AXIS/iPolloWork/blob/dev/LICENSE)

## 二、安装与系统

<a id="faq-011"></a>
### 011. iPolloWork 支持哪些电脑操作系统？

**简答：** 官方构建矩阵覆盖 Windows、macOS 和 Linux。

**范围 / 状态：** 桌面端 · Live

**依据：** [安装说明](https://github.com/Devin-AXIS/iPolloWork#install-ipollowork)

<a id="faq-012"></a>
### 012. iPolloWork 支持 Intel、AMD 和 ARM 电脑吗？

**简答：** 支持。macOS 提供 Apple Silicon 与 Intel 构建，Windows 和 Linux 提供 x64 与 ARM64 构建；下载时需要同时匹配操作系统和 CPU 架构。

**范围 / 状态：** 桌面端 · Live

**依据：** [安装矩阵](https://github.com/Devin-AXIS/iPolloWork#download-the-desktop-app)

<a id="faq-013"></a>
### 013. 应该下载哪个安装文件？

**简答：** macOS 普通用户选择 DMG，Windows 选择 EXE，Linux 选择 AppImage；文件名中的 mac/win/linux 与 arm64/x64 必须与设备匹配。

**范围 / 状态：** 桌面端 · Live

**依据：** [安装矩阵](https://github.com/Devin-AXIS/iPolloWork#download-the-desktop-app)

<a id="faq-014"></a>
### 014. ZIP、tar.gz 和安装包有什么区别？

**简答：** DMG、EXE、AppImage 是多数用户的推荐入口；macOS ZIP 和 Linux tar.gz 主要是便携或更新产物，不是首选安装方式。

**范围 / 状态：** 桌面端 · Live

**依据：** [安装说明](https://github.com/Devin-AXIS/iPolloWork#download-the-desktop-app)

<a id="faq-015"></a>
### 015. macOS 上如何安装 iPolloWork？

**简答：** 打开下载的 DMG，然后把 iPolloWork 拖入 Applications；需要按 Intel 或 Apple Silicon 选择对应安装包。

**范围 / 状态：** macOS · Live

**依据：** [安装说明](https://github.com/Devin-AXIS/iPolloWork#download-the-desktop-app)

<a id="faq-016"></a>
### 016. Windows 安装时 SmartScreen 警告代表什么？

**简答：** 本地构建且未签名的安装包可能触发 Microsoft Defender SmartScreen。正式用户应优先使用官方 Release 安装包，并核对下载来源。

**范围 / 状态：** Windows · Live

**依据：** [安装说明](https://github.com/Devin-AXIS/iPolloWork#download-the-desktop-app)

<a id="faq-017"></a>
### 017. Linux AppImage 如何运行？

**简答：** 下载与 CPU 架构匹配的 AppImage，赋予执行权限后运行；tar.gz 也可解压后便携运行。

**范围 / 状态：** Linux · Live

**依据：** [安装说明](https://github.com/Devin-AXIS/iPolloWork#download-the-desktop-app)

<a id="faq-018"></a>
### 018. 从源码开发 iPolloWork 需要什么环境？

**简答：** 基础要求包括 Git、Node.js 22 或更高版本、pnpm 11、Bun 1.3.10 或更高版本，以及各平台对应的原生构建工具链。

**范围 / 状态：** 开发环境 · Live

**依据：** [源码开发要求](https://github.com/Devin-AXIS/iPolloWork#requirements-for-source-development-and-packaging)

<a id="faq-019"></a>
### 019. 从源码启动 iPolloWork 的标准命令是什么？

**简答：** 克隆仓库并启用 Corepack 后，macOS/Linux 使用 ./ipollowork setup 与 ./ipollowork dev，Windows PowerShell 使用对应的 ipollowork.cmd 命令。

**范围 / 状态：** 开发环境 · Live

**依据：** [从源码启动](https://github.com/Devin-AXIS/iPolloWork#start-from-source)

<a id="faq-020"></a>
### 020. 开发模式会覆盖用户正常的 OpenCode 配置吗？

**简答：** 不会。项目说明明确指出开发模式使用隔离的 iPolloWork/OpenCode 状态，不覆盖用户的正常 OpenCode 配置。

**范围 / 状态：** 开发环境 · Live

**依据：** [从源码启动](https://github.com/Devin-AXIS/iPolloWork#start-from-source)

## 三、Work

<a id="faq-021"></a>
### 021. Work 在知识库中代表什么？

**简答：** Work 是办公与通用任务方向，覆盖文件、文档、演示文稿、网页操作、研究和会话工作流。

**范围 / 状态：** 官网信息架构 · 待确认命名

**依据：** [产品说明](/start-here/get-started)

<a id="faq-022"></a>
### 022. iPolloWork 可以直接处理本地文件吗？

**简答：** 可以。工作区围绕本地文件和仓库运行，Agent 可在授权范围内读取、创建和修改文件，结果继续保留在工作区。

**范围 / 状态：** 本地工作区 · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-023"></a>
### 023. iPolloWork 可以创建和修改文档吗？

**简答：** 可以，文档是产品明确支持的结果类型之一；具体格式和可编辑方式由当前任务、工具和文件类型决定。

**范围 / 状态：** Work · Live

**依据：** [开始使用](/start-here/get-started)

<a id="faq-024"></a>
### 024. iPolloWork 可以制作演示文稿吗？

**简答：** 可以。演示文稿属于可生成并继续编辑的结果类型，也可通过 Create 模板体系复用幻灯片设计。

**范围 / 状态：** Work、Create · Live

**依据：** [开始使用](/start-here/get-started)

<a id="faq-025"></a>
### 025. iPolloWork 能处理电子表格吗？

**简答：** Agent 可以处理工作区中的表格文件和数据，但官网不应承诺所有电子表格格式都具有相同的可视化编辑深度；应按具体文件格式和当前版本说明。

**范围 / 状态：** Work · 能力依格式而定

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-026"></a>
### 026. iPolloWork 可以自动操作网页吗？

**简答：** 可以。内置 iPolloWork Browser 支持打开网页、点击、填写表单、读取页面内容和截图。

**范围 / 状态：** 内置浏览器 · Live

**依据：** [控制浏览器](/start-here/do-work-with-it/control-the-browser)

<a id="faq-027"></a>
### 027. 浏览器控制等于控制整台电脑吗？

**简答：** 不等于。当前文档明确限定为 iPolloWork 内置浏览器中的 computer-use，不代表可以完整控制 Windows、macOS 或 Linux 的所有桌面应用。

**范围 / 状态：** 内置浏览器 · Live，桌面全控未承诺

**依据：** [控制浏览器](/start-here/do-work-with-it/control-the-browser)

<a id="faq-028"></a>
### 028. 什么是 iPolloWork Workflows？

**简答：** Workflows 是利用聊天、会话状态和分组把日常任务组织成可重复流程的方式，不是强制用户采用固定项目管理模型。

**范围 / 状态：** 会话管理 · Live

**依据：** [Workflows](/start-here/do-work-with-it/workflows)

<a id="faq-029"></a>
### 029. 会话可以怎样整理？

**简答：** 会话可以重命名、置顶、归档、排序或移动到工作区内的自定义分组；Agent 也可以按用户指令执行这些组织动作。

**范围 / 状态：** 会话管理 · Live

**依据：** [Workflows](/start-here/do-work-with-it/workflows)

<a id="faq-030"></a>
### 030. iPolloWork 有跨会话长期记忆吗？

**简答：** 当前跨会话能力来自已保存的会话历史，不是独立的长期记忆数据库，也不会自动把所有旧聊天注入每个新会话。

**范围 / 状态：** 会话历史 · Live，能力有限

**依据：** [跨会话记忆](/start-here/do-work-with-it/cross-chat-memory)

## 四、Code

<a id="faq-031"></a>
### 031. iPolloWork 可以打开现有代码仓库吗？

**简答：** 可以。产品以工作区为边界，可围绕已有仓库和本地文件运行 Agent 任务。

**范围 / 状态：** Code · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-032"></a>
### 032. Agent 可以读取和修改项目代码吗？

**简答：** 可以，但修改范围受当前工作区和权限审批约束；用户可以检查修改结果并继续编辑。

**范围 / 状态：** Code · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-033"></a>
### 033. iPolloWork 可以运行终端命令吗？

**简答：** 可以。Agent 执行流程包含运行命令，桌面界面也包含会话终端能力；高风险操作仍应经过权限控制。

**范围 / 状态：** Code · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-034"></a>
### 034. iPolloWork 只支持 TypeScript 项目吗？

**简答：** 不是。iPolloWork 自身主要使用 TypeScript，但 Agent 工作区不限定用户项目语言；实际能力取决于本机或远程 Worker 已安装的工具链。

**范围 / 状态：** Code · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-035"></a>
### 035. iPolloWork 能执行 Git 操作吗？

**简答：** 可以通过 Agent 和终端使用工作区现有的 Git，但提交、推送和破坏性操作应按用户授权和仓库流程执行。

**范围 / 状态：** Code · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-036"></a>
### 036. Agent 修改代码前会不会直接获得所有权限？

**简答：** 不应这样理解。iPolloWork 的产品定位包含用户检查与批准操作，实际权限由运行环境、工作区边界和具体授权策略共同决定。

**范围 / 状态：** Code、权限 · Live

**依据：** [项目 README](https://github.com/Devin-AXIS/iPolloWork)

<a id="faq-037"></a>
### 037. Code 任务必须在本机运行吗？

**简答：** 不必须。任务既可以在本地工作区运行，也可以连接自托管的远程 iPolloWork Server 或 Cloud Shared Workspace。

**范围 / 状态：** 本地与远程 · Live

**依据：** [共享工作区](/cloud/run-in-the-cloud/shared-workspace)

<a id="faq-038"></a>
### 038. 远程工作区适合什么开发任务？

**简答：** 适合需要稳定在线环境、长时间运行、团队共享或消息连接器的任务；远程环境中可用的语言和工具仍取决于 Worker 配置。

**范围 / 状态：** 远程 Worker · Live；Cloud Shared Workspace 为 Alpha

**依据：** [共享工作区](/cloud/run-in-the-cloud/shared-workspace)

<a id="faq-039"></a>
### 039. iPolloWork 自己的开发检查命令是什么？

**简答：** 仓库提供统一的 check 命令执行类型检查和桌面相关测试；开发者应使用 pnpm 和项目包装命令，而不是 npm 或 yarn。

**范围 / 状态：** iPolloWork 开发 · Live

**依据：** [开发命令](https://github.com/Devin-AXIS/iPolloWork#development-commands)

<a id="faq-040"></a>
### 040. iPolloWork 是否修改或分叉了 OpenCode？

**简答：** 没有。OpenCode 作为独立 sidecar，通过受支持的 API、CLI、插件和配置与 iPolloWork 集成，可独立升级。

**范围 / 状态：** 架构边界 · Live

**依据：** [架构边界](https://github.com/Devin-AXIS/iPolloWork#architecture-boundary)

## 五、Create

<a id="faq-041"></a>
### 041. Create 可以制作哪些内容？

**简答：** Create 可覆盖网站、应用原型、幻灯片、海报、卡片、报告、文章及其他设计类结果，最终能力取决于当前模板和编辑表面。

**范围 / 状态：** Create · Live

**依据：** [构建模板](/start-here/do-work-with-it/build-a-template)

<a id="faq-042"></a>
### 042. Create 生成的是图片还是可编辑页面？

**简答：** 设计模板推荐使用结构化 HTML、CSS 和资源文件，保留文本、布局和样式的可编辑性，而不是把整个结果压成单张图片。

**范围 / 状态：** Create · Live

**依据：** [构建模板](/start-here/do-work-with-it/build-a-template)

<a id="faq-043"></a>
### 043. Create 的网站和设计必须使用特定前端框架吗？

**简答：** 模板契约支持自包含的静态 HTML、CSS、JavaScript 和资源文件，不要求必须使用某个前端框架。

**范围 / 状态：** Create 模板 · Live

**依据：** [构建模板](/start-here/do-work-with-it/build-a-template)

<a id="faq-044"></a>
### 044. 什么是 Template Market？

**简答：** Template Market 用于安装和复用设计或视频起点。导入后模板进入个人库，使用模板时会为当前会话创建独立副本。

**范围 / 状态：** Create、Video · Live

**依据：** [构建模板](/start-here/do-work-with-it/build-a-template)

<a id="faq-045"></a>
### 045. iPolloWork 模板使用什么文件格式？

**简答：** 当前本地模板格式是 .ipwt，本质上是包含 manifest.json、入口文件、封面和资源的 ZIP 归档。

**范围 / 状态：** 模板开发 · Live

**依据：** [模板包结构](/start-here/do-work-with-it/build-a-template)

<a id="faq-046"></a>
### 046. 模板市场支持哪些分类？

**简答：** 当前契约包含 site、video、app、slides、poster、cards、report、article 和 other 九类。

**范围 / 状态：** Create、Video · Live

**依据：** [模板分类](/start-here/do-work-with-it/build-a-template)

<a id="faq-047"></a>
### 047. 导入模板有大小限制吗？

**简答：** 当前限制为上传包 50 MB、解压后 200 MB、最多 1,000 个文件、单文件最多 25 MB，并拒绝符号链接、可执行文件和路径穿越。

**范围 / 状态：** 模板导入 · Live

**依据：** [模板包结构](/start-here/do-work-with-it/build-a-template)

<a id="faq-048"></a>
### 048. 卸载模板会删除已经创建的作品吗？

**简答：** 不会。应用模板时会创建会话拥有的副本，卸载个人模板库中的模板不会删除既有会话作品。

**范围 / 状态：** Template Market · Live

**依据：** [构建模板](/start-here/do-work-with-it/build-a-template)

<a id="faq-049"></a>
### 049. Create 支持哪些导出格式？

<a id="faq-091"></a>
### 091. 模型和媒体 API Key 会发送到生成文件或 OpenCode 吗？

**简答：** 按当前本地能力契约，Media Center 等提供商凭据保存在本地 iPolloWork Server 边界，不返回给应用 UI、生成文件或 OpenCode。

### 092. Storage Center 会替代本地工作区文件吗？

**简答：** 不会。它只在功能明确需要时把一个工作区文件复制到对象存储，本地工作区文件、预览和下载仍保持本地优先。

**范围 / 状态：** Storage Center · Live

**依据：** [Storage Center](/start-here/do-work-with-it/storage-center)

<a id="faq-093"></a>
### 093. Storage Center 上传有什么安全和大小限制？

**简答：** sourcePath 必须位于当前工作区内且为普通文件，路径穿越会被拒绝；当前每次调用最多上传 100 MB，并且 API 不返回存储凭据。

**范围 / 状态：** Storage Center · Live

**依据：** [Storage Center](/start-here/do-work-with-it/storage-center)

<a id="faq-094"></a>
### 094. 第三方插件是否运行在完全隔离的安全沙箱中？

**简答：** 不能这样承诺。当前版本提供存储和 API 边界隔离，但原生 OpenCode 插件仍共享操作系统进程，不构成针对恶意原生插件的硬沙箱。

**范围 / 状态：** 插件安全 · 当前限制

**依据：** [插件授权与隔离](https://github.com/Devin-AXIS/iPolloWork/blob/dev/docs/plugin-packages.md)

### 097. 可以把 iPolloWork 做成 SaaS、白标产品或收费服务吗？

**简答：** 未经书面授权不可以。托管、SaaS、转售、付费客户交付、白标和其他商业使用均属于需要事先授权的范围。

**范围 / 状态：** 授权政策 · Policy

**依据：** [iPolloWork License](https://github.com/Devin-AXIS/iPolloWork/blob/dev/LICENSE)

<a id="faq-098"></a>
### 098. 企业可以私有化部署并使用 SSO 吗？

**简答：** 可以与团队讨论 VPC 或本地部署方案，包括 SSO、审计日志以及模型和工具控制；具体许可、架构和支持范围需要企业方案确认。

**范围 / 状态：** Enterprise · 需联系团队

**依据：** [Enterprise](/cloud/enterprise)

<a id="faq-099"></a>
### 099. 遇到问题时应该提供哪些信息？

**简答：** 提交 Issue 前应搜索重复问题，并提供 iPolloWork/OpenCode 版本、操作系统、复现步骤；桌面、Worker 或会话问题还应附运行时调试报告和开发者日志导出。

**范围 / 状态：** 支持流程 · Policy

**依据：** [GitHub Issues](https://github.com/Devin-AXIS/iPolloWork/issues)
