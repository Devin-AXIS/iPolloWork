---
title: Use Media Center
description: Use the provider-neutral Media Center contract for speech, translation, video, and digital-human capabilities without changing OpenCode.
---

iPolloWork exposes one local **Media Center** capability contract. Its first provider is Alibaba Model Studio and uses one `DASHSCOPE_API_KEY` from **Settings → Authorizations → Alibaba Model Studio media**. Provider code stays behind the local server: app UI, generated files, and OpenCode never receive the key value.

## What it provides

| Capability | Extension action | Result |
| --- | --- | --- |
| Text to speech | `speech_synthesize` | Audio URL from CosyVoice |
| Voice cloning | `voice_clone` | Reusable Model Studio voice ID |
| Recorded transcription | `speech_transcribe` | Asynchronous Fun-ASR task |
| Low-latency segment recognition | `speech_recognize_realtime` | Fun-ASR realtime response for a short accessible segment |
| Audio/video file translation | `speech_translate` | Translated text and optional base64 audio chunks |
| Video generation | `video_generate` | Asynchronous Wan task |
| Model-specific video edit | `video_edit` | Asynchronous Wan task |
| Digital human | `digital_human_generate` | Asynchronous Wan lip-sync task |
| Read task status | `task_get` | Provider task status and result URL |

Model Studio task result URLs are temporary. Use Storage Center when a feature needs to copy a workspace artifact to durable object storage.

Every Media Center call returns the same envelope: `provider`, `operation`, `output`, and `taskId` when the provider creates an asynchronous task. `output` preserves provider-specific fields without putting provider branching into the application call path.

## Call it from an iPolloWork app surface

Use the typed client helper. It always routes through the local iPolloWork server, never from the browser directly to Model Studio.

```ts
const result = await client.callMedia("speech_synthesize", {
  text: "欢迎来到 iPolloWork。",
  voice: "longanyang",
});

const translation = await client.callMedia("speech_translate", {
  fileUrl: "https://assets.example.com/meeting.wav",
  format: "wav",
  sourceLanguage: "zh",
  targetLanguage: "en",
  includeAudio: true,
});
```

For an asynchronous task, retain its `result.taskId` in the feature that created it, then poll deliberately instead of submitting duplicate work.

```ts
const task = await client.callMedia("task_get", { taskId });
```

Every input file must be an accessible HTTPS URL or a supported data URI. For production media, upload it to your own OSS bucket first and use a scoped, temporary URL.

## Call it from an agent

The bundled iPolloWork extension plugin exposes the same actions to the runtime. An agent first calls `ipollowork_extension_list_actions` and then calls `ipollowork_extension_call` with `extensionId: "media"`.

The agent-facing capability exists in iPolloWork only. It does not add a Model Studio provider to OpenCode, edit OpenCode source files, modify OpenCode configuration, or depend on OpenCode internal data. That separation keeps OpenCode upgrades straightforward.

## Upgrade check

After changing the bundled OpenCode version, verify the local bridge rather than patching OpenCode:

```bash
pnpm --filter ipollowork-server exec bun test src/extensions/media-center.test.ts src/extensions-connect-gating.test.ts
```

Then start the desktop development client, open a session, and confirm that `ipollowork_extension_list_actions` includes `media`.

## Realtime boundary

`speech_recognize_realtime` is designed for short, uploaded segments. `speech_translate` consumes Model Studio's streaming response and returns the complete translation through the local gateway. A continuous microphone-to-microphone experience needs a dedicated WebSocket relay with short-lived credentials; keep that relay inside iPolloWork when it is added, rather than exposing the permanent Model Studio key or changing OpenCode.
